## Figure 1 (Submission): Composite Results Figure

**Purpose:** Show sample efficiency and interpretable learned weights in a single full-width figure. For a 2-page extended abstract, one strong composite figure communicates more than separate method/results figures.

**Layout:** Two panels (A–B), full page width (`\textwidth`). Height ~55mm.

**Font:** Sans-serif, 7–8pt labels, 9pt panel letters.

---

### Panel A — Sample Efficiency Curves (width: ~62% of `\textwidth`)

**Plot type:** Line plot with shaded error bands (±1 std).

**X-axis:** $n$/class, log scale: {20, 50, 100, 200, 500, 1000}. Label: "Training samples per class".

**Y-axis:** Macro F1 (0–1.0). Grid lines at 0.33 (chance, red dashed) and 0.67 (symmetric ceiling, grey dotted).

**Lines (9 models), grouped by colour temperature:**

| Model | Colour | Role |
|---|---|---|
| SPI-MPNN | #D32F2F (red) | Full method |
| Fixed-SPI | #FF7043 (orange) | Vocabulary without learned topology |
| MLP-Mix | #FFB300 (amber) | Nonlinear adjacency |
| SGC-Only | #8D6E63 (brown) | Oracle-best single SPI |
| Edge-Ablation | #AB47BC (purple) | Topology only, no edge features |
| Correlation | #1976D2 (blue) | Single symmetric statistic |
| Shuffled | #78909C (blue-grey) | Broken pair correspondence |
| Latent | #7B1FA2 (dark purple) | Learned embeddings, no SPI |
| Node-Only | #9E9E9E (grey) | No pairwise features |

**Legend:** Compact, `ncol=3, fontsize=5`. Warm colours = SPI-consuming, cool = controls.

**Data source:** `results/sample_efficiency_definitive_w60_cpu_results.json` + SGC-Only results.

**Generated by:** `visualization.py → plot_sample_efficiency()`

**Output:** `figures/fig2a_sample_efficiency.pdf`

## Future Figures (Full Paper / Presentation)

Ideas worth exploring for a longer submission or talk. Not currently implemented.

### Method Schematic (Pipeline Diagram)

**Concept:** Full left-to-right pipeline: Raw MTS → pyspi → K layered M×M graphs compressing into $\mathbf{E}_{ij} \in \mathbb{R}^K$ edge vectors → learned $\mathbf{w}$ → sparse directed graph → MPNN → classification.

**Key visual element:** The K layered graphs (stacked semi-transparently) collapsing into a single edge vector — shows how 125 statistics become one number per edge. Annotate symmetric vs directed layers.

**Why it's interesting:** This is the conceptual diagram reviewers need to "get" the method in 5 seconds. For 2 pages the equations suffice, but for a poster/talk this is essential.

**Production:** Would need Inkscape/Illustrator. Data panels from Python, composed in vector editor.

### Symmetric vs Directed Contrast Grid

**Concept:** 2×2 or 3×2 grid of M×M heatmaps. Rows: chain instance, fork instance. Columns: Pearson |r| (symmetric, visually identical), SGC (directed, visibly different).

**Why it's interesting:** Makes the Markov equivalence argument visceral — "these look identical to Pearson, different to SGC." Strongest single visual for the theoretical claim.

**Data source:** Load one chain + one fork instance, extract Pearson and SGC matrices.

### SPI Weight Evolution Over Training

**Concept:** Animated or multi-frame plot showing how $\mathbf{w}$ evolves during training. Family-level L2 norms over epochs. Causal family should emerge as dominant after warmup period.

**Why it's interesting:** Shows the model "discovering" that causal statistics matter. Compelling for talks.

### Confusion Matrix Comparison

**Concept:** Side-by-side confusion matrices at n=100 for SPI-MPNN vs Correlation. SPI-MPNN should be near-diagonal; Correlation should show chain↔fork confusion (the Markov equivalence failure mode).

**Why it's interesting:** Directly shows WHERE symmetric methods fail — they confuse chain and fork, exactly as predicted.
